# Project LIFE Backlog

## Roadmap

1. [x] PC・スマホ同期
2. [ ] スマホUI完成
3. [ ] 生活ルーティン
4. [ ] テンプレート強化
5. [ ] 実績・習慣管理

## Backlog

### スマホUI改善

- 今日の予定を迷わず確認・実行できる表示へ磨き込む。
- 3日表示の閲覧・編集・ドラッグ操作を改善する。
- ホーム画面から起動した際の操作性を確認する。

### 生活ルーティン

- 仕事、食事、お風呂など、関連する予定を一つの流れとして扱う。
- 固定・連動・柔軟な予定の配置ルールを設計する。
- 親予定の変更に応じた安全な自動調整を実装する。

### テンプレート強化

- 生活パターンごとのテンプレートを作成・管理しやすくする。
- テンプレート適用前に内容を確認できるようにする。
- 重複や既存予定を壊さずに適用できるルールを整える。

### 予定チェック機能

- 予定を未実行・実行中・完了・スキップとして記録できるようにする。
- スマホから少ない操作で状態を更新できるようにする。

### 学習時間・作業時間集計

- カテゴリごとの予定時間と実績時間を集計する。
- 週単位で学習・作業の傾向を振り返れるようにする。

### 通知・リマインダー

- 次の予定や開始時刻を適切なタイミングで通知する。
- 通知しすぎず、行動の迷いを減らす設計にする。

### PWA改善

- ホーム画面起動、アイコン、standalone表示を各端末で確認する。
- 必要性を確認したうえで、オフライン表示や更新制御を検討する。

### あめモード

- 目的、利用場面、通常モードとの違いを整理する。
- 仕様を確定してから実装範囲と優先順位を決める。
